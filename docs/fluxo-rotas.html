Fluxo de rotas — ae3/auth-security

Referência visual passo a passo. Siga a ordem de cima para baixo.

1
Autenticação (sem token)
POST
/api/auth/register
Cria conta + retorna token
→ RETORNA access_token + user
ou
POST
/api/auth/login
email + password
→ RETORNA access_token + mfa_enrolled
Authorization: Bearer {access_token}
Use este header em todos os requests seguintes
2
Verificar se tem fator cadastrado
mfa_enrolled = true?
NÃO — Cadastrar fator
GET
/api/auth-security/mfa/contacts
Lista e-mails/telefones disponíveis
→ RETORNA [{ channel, identifier, label }]
↓
POST
/api/auth-security/mfa/factors
{ type, identifier, name }
type: otp_email | otp_sms | authenticator_app
→ RETORNA factor_id (pendente)
OTP cai no laravel.log (sandbox)
↓
POST
/api/auth-security/mfa/factors/{id}/confirm
{ code: "123456" }
→ Fator ativo — confirmed_at preenchido
↓
Fator cadastrado — agora siga o caminho "SIM" abaixo
SIM — Verificar MFA
GET
/api/auth-security/mfa/factors
Lista fatores confirmados do usuário
→ RETORNA [{ id, type, masked_identifier }]
↓
POST
/api/auth-security/mfa/factors/{id}/challenge
Dispara OTP ou instrui abrir app TOTP
→ CÓDIGO ENVIADO (OTP) ou instrução (TOTP)
Reenviar: POST .../challenge/resend
↓
POST
/api/auth-security/mfa/verify
{ factor_id, code }
→ RETORNA mfa_session_token + expires_at
X-Mfa-Session-Token: {mfa_session_token}
Adicionar este header nas rotas protegidas por MFA
3
Acesso a rotas protegidas
✓
Acesso liberado
Authorization: Bearer {access_token}
X-Mfa-Session-Token: {mfa_session_token}
!
Sem acesso ao fator? — Opções de recuperação
Via recovery code
POST
/api/auth-security/mfa/recovery-codes/verify
{ code: "XXXX-XXXX-XXXX" }
→ RETORNA mfa_session_token
Code invalidado após uso
ou
Via recuperação assistida (admin)
POST
/api/auth-security/mfa/assisted-recoveries
{ target_user_id, reason_category }
→ Admin libera token → usuário usa POST .../complete
Exige aprovação manual do admin
